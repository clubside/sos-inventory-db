# Code of Conduct

This project exists to build reliable tools, clear documentation, and a predictable ingestion workflow. To keep development productive and focused, all contributors are expected to follow a few simple rules of conduct.

## 1. Stay Technical

Issues, discussions, and pull requests must stay focused on the code, the schema, the ingestion pipeline, or the documentation. Personal arguments, unrelated debates, or attempts to derail technical threads are not allowed.

## 2. Be Direct, Not Disruptive

Clear, blunt communication is fine. Harassment, bad‑faith arguing, or intentionally wasting maintainers’ time is not.

Examples of disruptive behavior:

- Demanding immediate fixes or support
- Reopening closed topics without new information
- Arguing for the sake of arguing
- Posting inflammatory or hostile comments

## 3. Respect Maintainers’ Time

This is an open‑source project, not a paid support desk. Maintainers decide:

- what gets worked on
- when it gets worked on
- whether a feature or change fits the project

Entitlement or pressure will result in the discussion being closed.

## 4. No Drama

If a conversation becomes heated, unproductive, or off‑topic, maintainers may:

- lock the thread
- redirect the discussion
- close the issue or PR

This is not personal; it’s to keep the project moving.

## 5. Follow the Contribution Process

All contributors must:

- follow the documented coding standards
- write clear commit messages
- keep PRs focused and scoped
- update documentation when behavior changes

Ignoring the process creates unnecessary work for maintainers.

## 6. Enforcement

Maintainers may take any action needed to keep the project healthy, including:

- issuing a warning
- locking or closing threads
- rejecting or reverting contributions
- temporarily or permanently banning a user from participating

These actions are taken only to protect the project’s stability and productivity.

## 7. Final Note

This project welcomes contributors who want to build something solid, reliable, and future‑proof. If you keep discussions technical, respect the workflow, and avoid drama, you’ll fit right in.
